# Full-Site Redesign Contract

This project is not asking Codex to fill in a scaffold or lightly polish an existing page. Codex is the lead product designer and frontend engineer for the generated site.

## Authority

Codex may rewrite the entire generated frontend when quality requires it:

- `app/page.tsx`
- `app/globals.css`
- `app/layout.tsx`
- new files under `app/` or `components/`
- `package.json`, if a dependency is genuinely needed and the project remains buildable

The existing premium baseline is only a reference and fallback. It is not a design cage. If the baseline looks scaffold-like, replace it completely.

## Target quality

The result should look like a custom site from a small premium studio, not an AI-generated local-business template.

Aim for:

- an art-directed first viewport;
- a clear brand concept;
- strong typography and spacing;
- varied section rhythm;
- real visual hierarchy;
- intentional use of images, logo, and brand colour;
- high-quality mobile layout;
- useful interactions only when they fit the business;
- no generic card-stack homepage.

## Mandatory full-site design process

After plan mode, Codex must perform one full-site build pass. In that pass, Codex should:

1. choose a single visual concept for the whole site;
2. rewrite the page structure around that concept;
3. build all sections in one coherent design system;
4. create component-level variety without fragmenting the brand;
5. make the homepage feel complete, premium, and conversion-oriented;
6. delete any scaffold/baseline code that weakens the result.

## Premium visual patterns to prefer

Use one or two of these when appropriate:

- oversized editorial hero with disciplined negative space;
- split canvas with image, map, service menu, or contact panel;
- magazine-style section transitions;
- asymmetric bento with no empty holes;
- sticky left narrative with scrolling right cards;
- full-bleed photo treatment with refined overlay;
- restrained luxury palette rather than bright SaaS gradients;
- high-contrast final CTA panel;
- mobile sticky action bar;
- local proof/contact panel that feels designed, not pasted.

## What must be removed

Remove these even if the current scaffold contains them:

- placeholder-looking sections;
- repeated identical cards;
- generic headings such as “Services presented with clarity, not filler”;
- vague AI copy;
- centred hero plus three cards unless the design truly requires it;
- cheap shadows and random gradients;
- labels like `SECTION 01`, `ABOUT US`, or `QUESTION 05`;
- pages that could fit any business by changing only the name.

## Constraint

Do not invent facts. Expensive design should come from composition, typography, spacing, imagery, and interaction—not fake reviews, fake awards, fake certifications, fake years in business, or fake guarantees.
